about this data

The Response Formatter agent calls a single native ADK tool, save_incident_ticket, which assembles the prior agents’ session state into one INSERT INTO incident_tickets … RETURNING ticket_id against AlloyDB. Tickets are append-only; the workspace card the operator sees is rendered from the same row you’re looking at here.

What you’re looking at

ticket_id
Auto-incrementing PK. Mirrored into the workspace UI as “Ticket #N”.
category
Coarse classification — network, billing, device, … — emitted by the Classifier agent.
region
City-level metro inferred by the Classifier from the complaint text.
description
Verbatim customer complaint. Persisted exactly as received.
related_events
Comma-separated event_id values surfaced by the Network Investigator. Joins back into BigQuery for forensic context.
cdr_findings
Free-text rollup of what the CDR Analyzer found in AlloyDB — counts of failed / dropped calls in the window of interest.
recommendation
Final NOC-facing action plan synthesised by the Response Formatter. Drives the workspace action-chip strip.
status
open · resolved. Currently set by the agent on insert; manual transitions are out of scope.
created_at
UTC insert time. Default ordering for the viewer.
