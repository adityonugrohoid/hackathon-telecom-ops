about this data

The CDR Analyzer agent poses one English question to the MCP toolbox tool query_cdr_nl, which forwards it to alloydb_ai_nl.execute_nl_query('netpulse_cdr_config', $1). AlloyDB AI translates the question to SQL using a registered Gemini model, executes it, and returns rows under the read-only role netpulse_nl_reader.

What you’re looking at

call_id
Synthetic CDR identifier (PK). Stable across reseeds via deterministic generator.
caller_number
Originating MSISDN. Anonymised — safe to display in viewers.
receiver_number
Terminating MSISDN. Same anonymisation.
call_type
voice · sms · data. Drives volumetric rollups in the analyzer.
duration_seconds
Voice / data session length. Zero for failed setups and SMS messages.
data_usage_mb
Data volume for data calls; zero otherwise.
call_date
Session start in UTC. CDR Analyzer joins this against started_at windows from the Network Investigator.
region
City-level metro derived from the originating cell tower.
cell_tower_id
Identifier of the originating tower. Useful for narrowing post-incident root-cause analysis.
call_status
completed · dropped · failed. Failed/dropped clusters around outage windows by design in the seed.
