Telecom incident triage in seconds.

A multi-agent assistant that turns a customer complaint into a structured NOC incident ticket. Built with Google ADK, Vertex AI Gemini 2.5 Flash, BigQuery, and AlloyDB.

Launch NetPulse AI → View on GitHub
  • 4 ADK sub-agents orchestrated by a SequentialAgent root
  • BigQuery network events · AlloyDB call records · AlloyDB ticket sink
  • Vertex AI failover ladder: global → asia-southeast2 → asia-southeast1 → us-central1

How it works

Four sequential agents, each with a single job. The output of one becomes the input of the next.

  1. 1

    Classifier

    Reads the complaint and returns category + region. Native Python tool, no external I/O.

    NATIVE PYTHON TOOL
  2. 2

    Network Investigator

    Queries BigQuery for active outages in the affected region via the MCP Toolbox.

    BIGQUERY MCP TOOLBOX
  3. 3

    CDR Analyzer

    Inspects AlloyDB call records to confirm or rule out the customer's experience.

    ALLOYDB READ
  4. 4

    Response Formatter

    Synthesises the findings into a structured incident ticket and writes it to AlloyDB.

    ALLOYDB WRITE
